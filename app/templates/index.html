A FastAPI-based backend service for managing product and category catalogs with search, pagination, and production-oriented observability.

OpenAPI Docs ReDoc Health Check

Engineering Principles

  • Layered design with clear separation of API, service, and persistence concerns.
  • Schema-first contracts through Pydantic models and explicit validation rules.
  • Reliable data integrity via PostgreSQL constraints and transactional operations.
  • Observability-first implementation with structured logs, traces, and metrics.
  • Automated test coverage for core API behavior and product search scenarios.

Key Capabilities

  • /api/v1/categories: Create, update, list, and delete category trees.
  • /api/v1/products: CRUD operations for product catalog entries.
  • /api/v1/search/products: Filter products by title/SKU, price range, and category subtree.

The application is designed for maintainability and extension, allowing additional filters, fields, and operational controls to be added safely over time.
